# soundmeout(떠보기) - 익명 메세지 전달 플랫폼

### 떠보기는 누군가의 마음을 떠본다는 뜻도 있지만 떠나보내기의 약자이기도 합니다.

<br/>

살아가면서 졸업, 퇴사, 이직, 기념 등 다양한 이벤트들이 발생하고는 합니다. <br/>
그리고 우리는 이런 이벤트를 겪으며 가까운 누군가를 보내주고 또 환영합니다.<br/>
<br/>
그럴 때 마다 **자연스럽게 이용할 수 있는** 서비스를 기획하고 싶었습니다.<br/>
<br/>
떠나간 혹은 새로운 누군가에게 말하고 싶었지만 차마 하지 못한 말을 익명으로라도 전할 기회가 있다면 좋겠다고 생각하였습니다. <br/>
<br/>
**떠보기는 이를 가능하게 해주는 플랫폼입니다.**<br/>
<br/>
떠나보내기 서비스는 순간들을 추억으로 기록해 줄 수 있는 하나의 **추억상자**가 되어 주는 기능을 합니다.
이를 통해, 우리는 추억을 공유하고 서로를 위로할 수 있으며, 궁극적으로 떠나보내며 서비스는 지속가능한 서비스로 **좋은
영향력을 끼치는 서비스**가 되고자 합니다.

## 개인 목표
### 최성민
- 초당 1만건 이상의 요청 트래픽을 극복할 수 있는 설계 진행해보기.
- AWS를 이용한 배포와 CI/CD의 개념 정립하며 정리하기.
- 몽고디비의 스키마 모델링을 진행하되 인덱싱 목록부터 참조, 삽입의 유연한 도큐먼트 형식 제대로 활용하기.
### 이찬호
- 클라우드 컴퓨팅 중 가장 많이 사용되는 아마존 웹 서비스를 통해서 프로젝트를 배포하고 그 과정에서 CI/CD를 적용하여 배포환경 설정해보기
- 웹 프로젝트의 목적과 요구사항을 정확하게 이해하고, 이를 바탕으로 백엔드 개발의 기능과 구조를 파악해서 구현해보기.
- 협업에필요한 과정들을 경험하고 기술들을 익히면서 그 능력을 함양시키기.
### 고명준
- 프로젝트를 통해 웹 개발의 시작부터 배포까지 경험하고 이해하기.
- 협업에 필요한 기술(Git 등등)을 경험하고 습득하기.
- 클라이언트를 개발해 서버와의 통신과정, 전체 배포과정 까지 경험하고 습득하기.
- 단순히 만들기만 하는것이 아니라 성능최적화를 위한 작업까지 하기(캐싱 기술 등등)
- Client 쪽 역할을 맡음으로써 Server와 상호작용하기 위한 방법들을 습득.
- 전체 설계/개발과정에서 내가 부족한 점을 정확히 파악하고 한단계 성장하기

## 개선 방향
### git commit 메세지의 아쉬움
- 커밋 룰을 정했으나 오프라인에서 함께 진행하며 습관적으로 구두 전달 이후 의미없는 커밋 메세지 사용을 자주 진행함.
- 추후 프로젝트에서 개선 필요하며 관련한 예시를 많이 확인하고 적용하기

### git branch 전략
- 개발 중 merge, 배포의 과정에서 master branch가 아닌 develop branch가 main이 되어버림.
- git 전략에 대한 예시 많이 확인하고 적용하기

### 데이터베이스 선정과 모델링
- SNS성격의 플랫폼이며 모두에게 익숙한 MongoDB를 선정하였으나 막상 사용시 문서의 수정,변경 사항들이 다양했음.
- 서비스에서 데이터가 정확히 수정되지 않는 현상 발생. 인덱싱 문제로 추후 트랜젝션 및 인덱스 관련 재정비
